# SQL Query Reasoning

## Why this topic was chosen

SQL was chosen because it is one of the most common pain points for engineering students and one of the best fits for a grounded intervention system.

Students often struggle not because they know nothing, but because they make recurring reasoning mistakes:

- wrong join condition
- grouping mistakes
- misuse of `WHERE` vs `HAVING`
- duplicate counting
- incorrect subquery logic
- confusion around `NULL`

That makes SQL a strong first topic pack.

## Why it fits CogniWeave well

This topic works well for the project because:

- it has clear sub-skills
- query mistakes are easy to label
- improvement is measurable
- interventions can be highly specific

Examples of possible interventions:

- join-debugging drill
- predict-the-output exercise
- grouped aggregation trace
- rewrite-the-query task
- compare two similar queries and explain the difference

## What this pack proves

If the system works well on SQL, it shows that:

- the diagnosis step can identify structured reasoning mistakes
- the intervention step can map errors to specific corrective actions
- the evaluation step can use clear correctness and process signals

## Why it matters for engineering students

SQL is practical, placement-relevant, and common across coursework, internships, and project work.

It also gives the project a strong data-oriented angle without making it trivial.
